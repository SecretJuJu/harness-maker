# [Harness: rule-004] conventional-commits

All commit messages in this repo follow [Conventional Commits](https://www.conventionalcommits.org/): `type(scope)?: subject`.

- ✅ Good: `docs: switch README to English by default`, `chore: add .gitignore`, `feat(skill): add performance reference`, `fix!: rename rules.yaml field`.
- ❌ Bad: `update readme`, `fix stuff`, `wip`.
- Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `perf`, `ci`, `build`, `revert`.
- Exception: Merge, Revert, `fixup!`, and `squash!` commits are auto-generated by git and skipped.

Enforcement: `.harness/hooks/check-conventional-commits.sh` run by the `commit-msg` git hook validates the first line of each commit message.
